# 레트로 테트리스 재구축 명세

## 목표

모래 알갱이 시뮬레이션과 색상 연결 판정을 제거하고, 예측 가능하고 즉각적으로 조작되는 10×20 클래식 테트리스로 `/tetris`를 재구축한다. 시각 콘셉트는 KeyWork의 Win98/Mac-classic 셸 안에서 실행되는 1990년대 셰어웨어 아케이드 게임으로 통일한다.

## 디자인 방향

- **톤**: Win98 데스크톱 + 16비트 아케이드 캐비닛
- **기억점**: 검은 플레이필드, 원색 테트로미노, 픽셀 베벨, 라인 클리어 때 짧게 점멸하는 `TETRIS!` 전광판
- **원칙**: 레트로 외형은 강하게 유지하되 조작은 현대적으로 명확하게 제공한다.
- **금지**: 모래·지형 텍스처, 유기적 입자 시뮬레이션, 글래스모피즘, 과도한 화면 흔들림

## 사용자 시나리오

### P1 — 클래식 테트리스 플레이

- 사용자는 7종 테트로미노를 이동·회전·낙하시켜 완성된 가로줄을 제거한다.
- 블록은 격자에 정확히 고정되며 모래처럼 무너지거나 섞이지 않는다.
- 새 게임, 일시정지, 게임오버, 재시작 흐름이 키보드와 모바일에서 모두 동작한다.

### P1 — 예측 가능한 조작

- 좌우 이동, 소프트 드롭, 하드 드롭, 시계/반시계 회전이 즉시 반응한다.
- 고스트 피스로 착지 위치를 확인할 수 있다.
- 한 턴에 한 번 현재 피스를 홀드할 수 있다.
- 7-bag 출제 방식으로 특정 피스가 장시간 나오지 않는 문제를 방지한다.

### P1 — 점수와 진행

- 1/2/3/4줄 클리어 점수는 `100/300/500/800 × 레벨`이다.
- 소프트 드롭은 칸당 1점, 하드 드롭은 칸당 2점을 추가한다.
- 10줄마다 레벨이 상승하고 낙하 간격이 짧아진다.
- 게임오버 시 점수가 기존 로컬 개인 기록에 한 번만 저장된다.

### P2 — 레트로 피드백

- 라인 클리어는 짧은 행 점멸과 `SINGLE/DOUBLE/TRIPLE/TETRIS!` 텍스트로 알린다.
- 레벨 상승, 하드 드롭, 게임오버에는 짧고 절제된 효과를 사용한다.
- FX 비활성화 또는 모션 감소 설정에서는 흔들림과 플래시를 최소화한다.

## 기능 요구사항

### FR-01: 보드와 피스

- 보드는 10열 × 20행의 `Cell[][]`로 관리한다.
- I, O, T, S, Z, J, L 7종과 각 회전 상태를 지원한다.
- 고정 피스, 활성 피스, 고스트 피스는 별도 상태로 계산한다.

### FR-02: 회전과 충돌

- 벽·바닥·고정 블록 충돌을 셀 단위로 검사한다.
- 회전 시 `0, -1, +1, -2, +2` 순서의 단순 월킥을 적용한다.
- P1에서는 T-Spin, 백투백, 퍼펙트 클리어 보너스를 구현하지 않는다.

### FR-03: 입력

| 입력 | 동작 |
|------|------|
| ← / → | 좌우 이동 |
| ↓ | 소프트 드롭 |
| ↑ 또는 X | 시계 방향 회전 |
| Z | 반시계 방향 회전 |
| Space | 하드 드롭 |
| C | 홀드 |
| P | 일시정지 |

- 키 입력 중 페이지 스크롤을 방지한다.
- 모바일 버튼은 최소 44px 터치 영역과 명시적인 `aria-label`을 제공한다.

### FR-04: HUD

- 데스크톱: `HOLD | 10×20 BOARD | NEXT + SCORE/LINES/LEVEL` 3열 구성
- 모바일: 상단 압축 HUD, 중앙 보드, 하단 조작 패널
- 시작 화면 제목은 `RETRO TETRIS`로 표시한다.
- `/tetris` 메타데이터와 랜딩/사이드 메뉴 명칭은 모두 `테트리스 / Tetris`로 통일한다.

### FR-05: 렌더링

- CSS Grid 기반 10×20 보드를 셀 단위로 렌더링한다.
- 블록은 단색 면 + 밝은 상단/좌측 + 어두운 하단/우측의 픽셀 베벨을 사용한다.
- 정적 Tailwind 클래스로 블록 색과 베벨을 표현한다.
- 모래 그리드, 모래 시뮬레이션, 연결 그룹 탐색, 지형 텍스처는 제거한다.

### FR-06: 로컬 기록

- 기존 `useLocalScoreSubmit`과 `game_mode: "tetris"`를 유지한다.
- 0점은 저장하지 않고 게임 세션당 한 번만 저장한다.
- `/leaderboard`의 테트리스 기록과 호환한다.

## 비기능 요구사항

- 게임 루프는 매 프레임 React state를 대량 생성하지 않는다.
- 게임 규칙은 가능한 한 순수 함수로 분리해 단위 테스트할 수 있어야 한다.
- 한국어/영어 UI와 Win98/Mac-classic 테마를 모두 지원한다.
- 새 의존성을 추가하지 않는다.

## 범위 제외

- 온라인 대전, 로그인, 글로벌 랭킹
- T-Spin/백투백/퍼펙트 클리어 보너스
- 커스텀 키 바인딩, 리플레이, AI 플레이
- 모래 물리나 지형별 블록 능력

## 성공 기준

- 모래 물리 관련 런타임 코드와 `SANDTRIS` 문구가 0건이다.
- 7-bag, 충돌, 회전, 고정, 줄 삭제, 점수 계산 단위 테스트가 통과한다.
- 데스크톱·모바일에서 시작 → 플레이 → 일시정지 → 게임오버 → 재시작 흐름이 동작한다.
- `npm run lint`, `npm run test:run`, `npm run build`가 통과한다.
