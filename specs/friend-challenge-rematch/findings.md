# findings.md

## 2026-03-13 조사 메모

### 확인된 상태
1. `useGameInvite`와 `InviteToast`는 이미 존재하지만, 현재 `MultiplayerLobby` 안에서만 사용된다.
2. 친구 초대 패널은 로비 안에서만 열 수 있고, 프로필 친구 목록에서는 실제 대전 액션이 없다.
3. 온라인 게임 페이지는 내부 `mode` state로 싱글/온라인을 나누고 있어, 쿼리 기반 자동 전환을 추가할 수 있다.

### 구현 방향
- 전역 초대 호스트를 `AppFrame`에 두고 모든 게임/프로필 화면에서 초대를 받게 한다.
- 수락 시 쿼리 기반으로 온라인 게임 페이지로 이동시키고, 해당 페이지에서 자동 참가를 수행한다.
- 친구 목록은 게임 선택 상태를 갖고 도전 버튼으로 해당 흐름을 시작한다.

## 2026-03-13 구현 메모

### 반영 내용
1. `GlobalInviteHost`를 추가해 앱 전역에서 게임 초대 토스트를 띄우고, 수락 시 해당 게임의 온라인 화면으로 바로 이동하게 했다.
2. `tetris`, `word-chain` 페이지는 `online`, `invite`, `inviteTarget` 쿼리를 읽어 자동 온라인 전환, 방 참가, 방 생성 후 초대 전송을 수행하도록 바꿨다.
3. 기존 로비는 친구 초대 발송만 담당하고, 초대 수신 토스트는 전역 호스트로 옮겨 중복 표시를 제거했다.
4. 프로필 친구 목록에 게임 선택형 `친구 도전` 섹션과 각 친구별 `도전` 버튼을 추가했다.
5. `useSearchParams`를 사용하는 페이지는 `Suspense` 래퍼로 감싸 Next.js 15 빌드 제약을 충족시켰다.
