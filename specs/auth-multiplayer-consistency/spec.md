# spec.md

## 문제
1. `useAuth` 예외 경로가 `user`만 유지하고 `profile`은 비워 두어, 일부 UI는 로그인으로 보이고 프로필 페이지는 비로그인처럼 보일 수 있다.
2. `rooms` 테이블의 `UPDATE` 정책이 기존 참가자만 수정 가능하게 되어 있어, 새 참가자가 `player2_id`를 채우는 동작이 RLS에 막힌다.

## 목표
1. 인증 상태가 실패/지연 상황에서도 한 가지 기준으로만 노출되게 만든다.
2. 멀티플레이 방 참가가 현재 앱 플로우에 맞게 동작하도록 정책을 수정한다.
3. 회귀 테스트와 DB 확인으로 동일 증상이 다시 발생하지 않게 한다.

## 비목표
- 인증 방식 변경
- 멀티플레이 UX 리디자인
- 새로운 게임 모드 추가

## 수용 기준
1. `useAuth`가 프로필 복원 실패 시 로그인 UI와 프로필 UI가 서로 다른 상태를 보이지 않는다.
2. `rooms` 정책 수정 후 로그인한 다른 사용자가 `player2_id`를 설정할 수 있다.
3. 관련 테스트가 추가되거나 보강된다.
4. `npm run lint`, `npm run test:run`, `npm run build`가 통과한다.
