# findings.md

## 2026-03-13 조사 메모

### 확인된 상태
1. `useUserStats`가 이미 `game_scores` 전체를 조회해 모드별 통계, 멀티 승률, 최근 기록을 계산하고 있다.
2. 프로필 페이지에는 전적, 업적, 친구 섹션이 이미 존재해 리텐션 카드만 추가하면 된다.
3. 일일 미션과 스트릭은 별도 테이블 없이 `created_at`, `game_mode`, `is_multiplayer`만으로도 계산 가능하다.

### 구현 방향
- `useUserStats`의 집계 로직을 순수 함수로 분리해 테스트 가능하게 만든다.
- 오늘 플레이 수, 오늘 플레이한 모드 수, 오늘 멀티플레이 수를 일일 미션으로 사용한다.
- 최근 7일 활동과 현재/최고 스트릭을 같은 데이터에서 계산한다.

## 2026-03-13 구현 메모

### 반영 내용
1. `src/lib/userStats.ts`를 추가해 유저 전적 집계, 일일 미션, 최근 7일 활동, 현재/최고 스트릭 계산을 순수 함수로 분리했다.
2. `useUserStats`는 기존 `game_scores` 조회 결과를 새 집계 함수에 넘기도록 단순화했고, 모드 라벨은 실제 네비게이션 모드 기준으로 정렬되게 맞췄다.
3. 프로필 페이지에 `오늘의 루프` 카드를 추가해 스트릭 상태, 최근 7일 활동, 3개의 일일 미션 진행 바를 표시하도록 했다.
4. `TypingInput`이 `AuthProvider`를 요구하는 현재 구조에 맞춰 기존 스모크/폴백 테스트 모킹을 보정했다.
5. `tests/lib/userStats.test.ts`를 추가해 미션 진행도와 스트릭 계산을 회귀 테스트로 고정했다.
