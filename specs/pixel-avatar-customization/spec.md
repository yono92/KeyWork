# Pixel Avatar Customization — Spec

## 사용자 스토리

1. **US-1**: 사용자는 프로필 페이지에서 아바타 에디터를 열어 파츠를 조합하고 저장할 수 있다.
2. **US-2**: 사용자의 아바타는 사이드바, 리더보드, 멀티플레이 로비/대전 화면에 표시된다.
3. **US-3**: 멀티플레이 대전 시 상대방의 아바타를 볼 수 있다.
4. **US-4**: 아바타를 설정하지 않은 사용자는 기본 아바타(닉네임 첫 글자)가 표시된다.

## 아바타 파츠 구조

### 레이어 (아래→위 순서)
| 레이어 | 카테고리 | 옵션 수 | 필수 |
|--------|----------|---------|------|
| 1 | 피부색 (Skin) | 6 | O |
| 2 | 헤어 (Hair) | 10 | O |
| 3 | 눈 (Eyes) | 8 | O |
| 4 | 입 (Mouth) | 6 | O |
| 5 | 모자/머리장식 (Hat) | 8 | X |
| 6 | 악세서리 (Accessory) | 6 | X |

### 렌더링 방식
- 32x32 픽셀 그리드 기반 도트 아트
- 각 파츠는 CSS 또는 Canvas로 렌더링하는 **코드 기반 픽셀 매트릭스**
- 파츠 데이터: `number[][]` 배열 (0=투명, 1~N=팔레트 인덱스)
- 팔레트: 레트로 테마 색상 기반 제한 팔레트 (16색)

### 데이터 저장 형식
```typescript
interface AvatarConfig {
  skin: number;     // 0-5
  hair: number;     // 0-9
  hairColor: number; // 0-7
  eyes: number;     // 0-7
  mouth: number;    // 0-5
  hat: number;      // -1(없음) ~ 7
  accessory: number; // -1(없음) ~ 5
}
```
- `profiles.avatar_config JSONB` 컬럼으로 저장
- 기존 `avatar_url` 컬럼은 유지 (하위호환)

## 기능 요구사항

### FR-1: 아바타 에디터
- 프로필 페이지에 "아바타 편집" 버튼 추가
- 카테고리별 탭으로 파츠 탐색
- 파츠 선택 시 32x32 프리뷰 실시간 반영
- "저장" 시 DB 업데이트, "취소" 시 원복
- 레트로 테마(win98/mac-classic) 스타일 적용

### FR-2: 아바타 렌더링 컴포넌트 (`PixelAvatar`)
- `size` prop으로 표시 크기 조절 (sm=24px, md=32px, lg=48px, xl=64px)
- `config` prop으로 AvatarConfig 수신
- config가 null이면 닉네임 첫 글자 폴백
- 레이어 순서대로 픽셀 합성하여 단일 캔버스/div 출력

### FR-3: 표시 위치 통합
| 위치 | 사이즈 | 비고 |
|------|--------|------|
| 사이드바 로그인 영역 | sm (24px) | 닉네임 옆 |
| 프로필 페이지 | xl (64px) | 편집 버튼 옆 |
| 리더보드 순위 항목 | sm (24px) | 닉네임 앞 |
| 멀티 로비 대기 | lg (48px) | 내/상대 |
| 테트리스 배틀 상단 | md (32px) | 나/상대 라벨 옆 |
| 끝말잇기 채팅 말풍선 | sm (24px) | 말풍선 왼쪽 |

### FR-4: 멀티플레이 아바타 전송
- Presence 트래킹에 `avatar_config` 포함
- 상대 아바타를 로비/대전 화면에 렌더링

## 비기능 요구사항

- **NFR-1**: 아바타 렌더링은 60fps 유지 (Canvas 기반, DOM 노드 최소화)
- **NFR-2**: 파츠 데이터는 코드 내 정적 데이터 (이미지 파일 불필요)
- **NFR-3**: DB 마이그레이션은 `ALTER TABLE profiles ADD COLUMN avatar_config JSONB`만 필요
- **NFR-4**: 아바타 미설정 사용자에게 기존 동작(닉네임 첫 글자) 유지

## Acceptance Criteria

- [ ] AC-1: 프로필에서 아바타 에디터를 열고 모든 파츠 카테고리를 탐색할 수 있다
- [ ] AC-2: 파츠 변경 시 프리뷰가 실시간 갱신된다
- [ ] AC-3: 저장 후 새로고침해도 아바타가 유지된다
- [ ] AC-4: 사이드바, 리더보드, 프로필에 아바타가 표시된다
- [ ] AC-5: 멀티 대전 시 상대 아바타가 보인다
- [ ] AC-6: 아바타 미설정 시 닉네임 첫 글자 폴백이 동작한다
- [ ] AC-7: win98/mac-classic 양쪽 테마에서 UI가 정상이다
