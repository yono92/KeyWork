# findings.md

## 2026-03-03 랜덤 단어 다양성 개선

### 조사 결과
1. `TypingRunnerGame`, `FallingWordsGame`는 최근 단어 이력 없이 완전 랜덤 선택이었다.
2. `useKoreanWords`도 동일하게 완전 랜덤 선택이었다.
3. `WordChainGame`은 시작 글자 제약 기반 선택이 핵심이라 동일 이슈 성격이 달랐다.

### 결정 사항
- 최근 4개 단어 기준의 유사도 필터를 각 모듈에 동일 정책으로 적용한다.
- 한국어: 동일 단어 제외 + 최근 단어와 접두 2자 이상 중복 제외 + 직전 단어와 접두 1자 중복 제외.
- 영어: 동일 단어 제외 + 접두 3자 이상 중복 제외.
- 후보군이 비면 원본 후보군으로 폴백한다.

### 리스크
- 데이터 풀이 매우 작을 경우 폴백으로 인해 유사 단어가 다시 출제될 수 있다.
- 현재 정책은 접두 비교 기반이므로 발음/의미 유사도까지는 다루지 않는다.

### 수동 검증 시나리오
1. `/typing-runner` 한국어 플레이 시 연속 시작어 중복 빈도 확인
2. `/falling-words` 한국어 플레이 시 연속 시작어 중복 빈도 확인

### 검증 결과
- `npm run lint`: 통과 (에러 0, 기존 경고 1건 유지: `src/components/WordChainGame.tsx:249`)
- `npm run test:run`: 통과 (11 files, 35 tests)
- `npm run build`: 통과 (Next.js 15.5.12 production build)

### 추가 점검 메모
1. 끝말잇기 `WordChainGame`의 랜덤 선택은 체인 규칙(시작 글자 제약) 하의 후보 선택으로 본 이슈와 성격이 달라 이번 범위에서 제외했다.
2. 문장 랜덤(`TypingInput`, `sentenceUtils`)은 단어 출제 문제가 아니라 문장 연습 모드 로직이라 제외했다.
