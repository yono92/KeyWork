# findings.md

## 2026-03-12 조사 메모

### 확인된 상태
1. 현재 온라인 흐름은 목록 로비에서 create/join 후 본게임 시작 시점에만 전투 화면으로 이동한다.
2. `useMultiplayerRoom`은 참가 직후 자동 카운트다운을 시작하고 `Ready` 개념이 없다.
3. 테트리스는 즉시 플레이가 가능하지만 시작 시 리셋 개념이 없고, 끝말잇기는 대기 중 워밍업 UX가 없다.

### 구현 메모
- 게임룸 입장과 본게임 시작을 분리해 “입장 중”과 “플레이 중”을 다른 상태로 둔다.
- 본게임 시작은 `Ready` 완료 후에만 허용하고, 시작 순간 게임 상태를 초기화한다.

## 2026-03-12 구현 메모

### 반영 내용
1. `useMultiplayerRoom`에 `myReady`, `opponentReady`, `setReady()`를 추가하고, 호스트가 양측 `Ready` 완료 시 자동으로 카운트다운을 시작하도록 수정했다.
2. 방 참가 시 즉시 `status = waiting` 상태의 게임룸으로 들어가고, 실제 시작 전까지는 프리게임 룸 상태를 유지하게 했다.
3. 테트리스는 프리게임 동안 혼자 보드를 연습할 수 있고, 카운트다운 시작 시 보드/상대 상태를 리셋해 공식 대전을 새 판으로 시작한다.
4. 끝말잇기는 프리게임 동안 워밍업 단어 입력을 지원하고, 공식 대전 시작 시 메시지/점수/목숨/턴 상태를 초기화한다.
5. 공통 `RoomReadyPanel`을 추가해 두 게임 모두 같은 대기/Ready UX를 사용하게 했다.
