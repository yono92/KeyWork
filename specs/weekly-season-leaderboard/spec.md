# spec.md

## 문제
1. 현재 랭킹 페이지는 단순 리스트 형태라 경쟁 서비스의 중심 화면처럼 느껴지지 않는다.
2. 랭킹은 현재 점수 레코드 단위라 같은 유저가 여러 번 노출될 수 있어 시즌 순위 감각이 약하다.
3. 주간 시즌 기준의 남은 시간, 참가자 수, 내 위치가 보이지 않아 재방문 동기가 약하다.

## 목표
1. 랭킹을 유저별 최고 점수 기준으로 집계해 시즌 경쟁 화면답게 만든다.
2. 주간 시즌을 기본 탭으로 두고, 시즌 종료 카운트다운과 참가자 규모를 표시한다.
3. 내 현재 순위와 최고 점수를 보여줘 재도전 동기를 만든다.

## 비목표
- DB 테이블 추가
- 시즌 보상 지급
- 친구 전용 랭킹

## 수용 기준
1. 랭킹 페이지 기본 기간이 `이번 주`로 설정된다.
2. 같은 유저는 한 시즌/기간 내 최고 점수 1건만 랭킹에 반영된다.
3. 주간 시즌에서 남은 시간, 참가자 수, 내 순위 또는 미진입 상태를 볼 수 있다.
4. 상위 3명은 일반 리스트와 구분되는 상단 영역으로 표시된다.
5. `npm run lint`, `npm run test:run`, `npm run build`가 통과한다.
