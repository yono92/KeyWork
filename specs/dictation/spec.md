# 받아쓰기 — Spec

## 사용자 스토리

1. **게임 시작**: 사이드 네비에서 "받아쓰기" 선택 → 난이도 선택 화면.
2. **난이도 선택**: Easy/Normal/Hard → 게임 시작.
3. **문장 듣기**: TTS가 문장을 읽어줌. 재생 버튼(🔊)으로 다시 들을 수 있음.
4. **타이핑**: 들은 문장을 입력창에 타이핑. Enter로 제출.
5. **채점**: 제출하면 원문과 비교 → 정확도 표시. 맞은 글자/틀린 글자 색상 구분.
6. **다음 문제**: 채점 결과 확인 후 "다음" 버튼 → 다음 문장.
7. **결과**: 10문제 완료 후 총점, 평균 정확도, 리더보드 제출.

## 기능 요구사항

### FR-1: TTS 재생
- Web Speech API `speechSynthesis` 사용
- 한국어: `ko-KR` 음성, 영어: `en-US` 음성
- 난이도별 재생 속도: Easy 0.8×, Normal 1.0×, Hard 1.2×
- 난이도별 재생 횟수: Easy 3회, Normal 2회, Hard 1회
- 수동 재생 버튼: 언제든 다시 들을 수 있음 (제출 전까지)
- TTS 미지원 시 폴백: 문장을 2초간 보여준 후 숨김

### FR-2: 문제 시스템
- 1세트 = 10문제
- proverbs.json에서 중복 없이 선택 (getRandomSentenceUnique)
- 문제 번호 표시 (1/10, 2/10 등)

### FR-3: 타이핑 & 채점
- GameInput 재사용 (한국어 IME 지원)
- Enter로 제출
- 정확도: 한국어는 자모 단위(calculateHangulAccuracy), 영어는 문자 단위
- 채점 결과: 원문 표시 + 글자별 맞음(녹색)/틀림(빨간색) 하이라이트
- 제출 후에는 입력 불가 (다음 문제 대기)

### FR-4: 점수 시스템
- 문제당 점수: `정확도(0~100)` 그대로
- 총점: 문제별 점수 합산 (최대 1000점)
- 난이도 배율: Easy 0.7×, Normal 1.0×, Hard 1.5×
- 최종 점수 = `총점 × 난이도배율`

### FR-5: 결과 화면
- GameOverModal 재사용
- 표시: 최종 점수, 평균 정확도, 완벽 문제 수(100%), 난이도
- 리더보드 제출 (로그인 시)

### FR-6: 난이도
| 항목 | Easy | Normal | Hard |
|------|------|--------|------|
| TTS 속도 | 0.8× | 1.0× | 1.2× |
| 자동 재생 | 3회 | 2회 | 1회 |
| 점수 배율 | 0.7× | 1.0× | 1.5× |

## 비기능 요구사항

- **NFR-1**: TTS 미지원 브라우저 대응 (폴백 텍스트 노출)
- **NFR-2**: 기존 코드 패턴 준수
- **NFR-3**: 모바일 대응
