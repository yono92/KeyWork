# 관리자 대시보드

## 배경
Supabase DB가 연동되어 유저, 점수, 방, 업적 등의 데이터가 쌓이고 있으나,
이를 관리할 수 있는 관리자 페이지가 없음. 게임 난이도 등의 설정도 코드에 하드코딩되어 있어
변경 시 배포가 필요.

## 목적
- 관리자 전용 대시보드(`/admin`)를 통해 유저/점수/방 데이터 조회·관리
- 게임 설정(난이도 파라미터 등)을 DB 기반으로 관리하여 코드 배포 없이 조정 가능
- `profiles.role` 컬럼으로 관리자 권한 분리

## 범위
- profiles 테이블에 role 컬럼 추가
- 미들웨어에서 /admin 경로 접근 시 admin role 체크
- 관리자 대시보드 UI (유저 관리, 점수 관리, 방 관리, 게임 설정)
- game_config 테이블로 게임 난이도 파라미터 DB 관리

## 제외 범위
- 관리자 간 역할 세분화 (super_admin 등)
- 감사 로그 (audit log)
- 대시보드 분석/차트 (향후 확장)
