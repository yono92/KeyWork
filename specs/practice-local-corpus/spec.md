# spec.md

## 사용자 스토리
- 플레이어로서 문장연습이 외부 API 상태와 무관하게 즉시 시작되길 원한다.
- 운영자로서 속담/문장 데이터는 런타임과 분리된 수집 파이프라인으로 관리하고 싶다.

## 기능 요구사항
1. 문장연습은 위키피디아 API를 호출하지 않고 로컬 데이터셋만 사용해야 한다.
2. 문장연습은 언어 변경 및 Enter 제출 시 로컬 데이터셋 기준으로 다음 문장을 순환해야 한다.
3. 우리말샘 Open API 기반 수집 스크립트가 제공되어야 한다.
4. 공공데이터 파일(JSON/CSV/TXT) 병합 입력 경로를 지원해야 한다.
5. 수집 결과는 기존 런타임 데이터 파일 형식과 호환되어야 한다.

## 비기능 요구사항
1. 문장연습 첫 문장 로딩은 네트워크 없이 동작해야 한다.
2. 수집 스크립트는 인증키 미설정, 응답 오류, 잘못된 입력 파일에 대해 명시적으로 실패해야 한다.
3. 런타임 변경은 다른 게임 모드의 API 사용 방식에 영향을 주지 않아야 한다.

## Acceptance Criteria
1. `TypingInput` 렌더링 시 `/api/wikipedia` 호출 없이 로컬 문장이 표시된다.
2. `app/api/wikipedia` 제거 후 테스트/빌드가 깨지지 않는다.
3. `node scripts/sync-practice-corpus.mjs --help`가 실행 가능하다.
4. `npm run lint`, `npm run test:run`, `npm run build`가 모두 통과한다.
