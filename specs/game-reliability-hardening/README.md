# 게임 안정성 강화 (game-reliability-hardening)

## 개요
외부 API 의존 구간(국립국어원 사전, 위키백과)에서 발생하는 실패를 사용자 경험 저하 없이 처리하고, 장애 원인을 추적 가능한 상태로 만드는 기능 설계를 시작한다.

## 목적
- 네트워크/API 장애 시 게임이 즉시 멈추거나 의미 없는 실패 메시지를 노출하는 문제를 줄인다.
- 장애 상황에서도 플레이 가능한 폴백 경로를 제공한다.
- 실패율/응답시간/폴백 사용률을 기록해 품질 개선 루프를 만든다.

## 이번 스펙에서 다루는 범위
- `word-chain`, `falling-words`, `practice`의 외부 데이터 의존 구간
- `/api/krdict/*`, `/api/wikipedia` 서버 라우트의 실패 처리 규칙
- 사용자 안내 문구, 재시도 전략, 로컬 폴백 전략
- 테스트/검증 기준 수립

## 제외 범위
- 게임 룰 개편, UI 대규모 리디자인
- 신규 외부 서비스(모니터링 SaaS) 도입
- 데이터셋 대규모 교체

## 현재 확인된 잠재 문제
1. 외부 API 실패 시 게임별 대응 수준이 다르고 사용자 안내가 일관되지 않음.
2. API 라우트 캐시(`Map`)는 상한/정리 정책이 없어 장시간 운영 시 메모리 증가 위험이 있음.
3. API 라우트 단위 테스트가 없어 회귀를 조기에 잡기 어려움.
4. 실패 이벤트를 구조화해서 남기지 않아 장애 원인 분석이 어려움.
