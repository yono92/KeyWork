# spec.md

## 문제
1. 멀티플레이 로비가 전투 시작과 함께 언마운트되며 Realtime 채널이 끊길 수 있다.
2. self-broadcast 이벤트를 자기 메시지로 걸러내지 않아 테트리스 상태와 끝말잇기 턴이 꼬인다.
3. 방 이탈과 오래된 방 정리가 불완전해 유령 방이 남고 참가 실패가 반복될 수 있다.
4. 회원가입 후 프로필 생성 실패 시 Auth 계정과 앱 프로필이 어긋나 로그인 경험이 불안정해진다.

## 목표
1. 전투 시작 후에도 동일한 Supabase room/channel 컨텍스트를 유지한다.
2. 자기 broadcast는 무시하고 상대 이벤트만 반영하도록 payload 규약을 정리한다.
3. 방 이탈, 오래된 방 필터링, 정책 보강으로 유령 방 노출을 줄인다.
4. 회원가입 직후 프로필이 반드시 생성되도록 복구 경로를 둔다.

## 비목표
- 멀티플레이 UI 리디자인
- 새로운 인증 방식 추가
- 랭킹 규칙 변경

## 수용 기준
1. 온라인 대전 시작 후 로비 언마운트로 채널이 끊기지 않는다.
2. 테트리스가 자기 보드/가비지를 상대 이벤트로 오인하지 않는다.
3. 끝말잇기 선공이 실제 유저 id 기준으로 결정되어 교착이 발생하지 않는다.
4. 방 나가기와 오래된 방 처리 후 유령 방이 로비에 다시 노출되지 않는다.
5. 회원가입 뒤 프로필 생성 실패가 fallback으로 복구되며 회귀 테스트가 추가된다.
6. `npm run lint`, `npm run test:run`, `npm run build`가 통과한다.
