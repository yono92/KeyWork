# findings.md

## 2026-03-12 조사 메모

### 확인된 상태
1. 현재 프로필 페이지는 `Card` 하나 안에 요약, 닉네임 편집, 아바타 편집, 가입일이 모두 들어 있어 계층이 약하다.
2. 성공/실패 피드백이 하나의 작은 문장으로만 표현돼 상태 전달력이 낮다.
3. `user.email`, `profile.nickname`, `profile.avatar_config`, `profile.created_at`만으로도 더 풍성한 요약/정보 구성이 가능하다.

### 구현 방향
- 기존 레트로 스타일 변수는 유지하고, 페이지 안에서만 구조와 밀도를 높인다.
- 요약 패널, 편집 패널, 계정 정보 패널을 분리해 한눈에 이해되는 편집 화면으로 만든다.

## 2026-03-12 구현 메모

### 반영 내용
1. 프로필 페이지 상단을 `플레이어 카드 + 편집 가이드` 구조로 재배치해 첫 화면에서 프로필 요약과 편집 목적이 동시에 보이게 했다.
2. 닉네임 편집 영역을 별도 카드로 분리하고 현재 값, 글자 수, Enter/Escape 키 동작, 저장/취소 버튼을 함께 노출했다.
3. 아바타 편집을 전용 작업실 카드로 분리해 현재 아바타 요약과 실제 편집기를 나눠 보여 주도록 바꿨다.
4. 계정 정보, 현재 테마, UI 언어, 가입일을 별도 정보 카드로 분리해 이전보다 정보 밀도를 높였다.
5. 성공/실패 메시지를 배너 스타일로 분리하고, 아바타 저장 실패도 같은 방식으로 표시되게 정리했다.
